## 任务 1：summarize.py 强化「摘要 vs 编者按」分工

**文件**：`src/processors/summarize.py`

### 现状
- `summarize_item()` prompt 同时输出 `chinese_summary` 和 `editor_note`，但**没有规定字数和分工**，结果两者写法雷同 + chinese_summary 写到 200+ 字。

### 目标分工（用户已确认 = 思路 A）

| 字段 | 字数 | 人格 | 内容 |
|---|---|---|---|
| `chinese_summary` | **60-80 字，最多两句** | 客观新闻体 | 提炼"发生了什么"——核心事实 + 关键数字/参数 |
| `editor_note` | 80-120 字（不变） | 主观编辑视角 | 评价"为什么值得读 / 它有什么意味"——判断、对比、所谓"祛魅"角度 |

### 改动要点
找到 `summarize.py` 里通用 prompt（不是 `summarize_github_repo`，那个 GitHub 专用 prompt 里 GitHub 源不写 `chinese_summary` / `editor_note`）。在 schema 描述里给两个字段加**字数硬约束**和**人格描述**。建议措辞类似：

```
- chinese_summary: 60-80 字的客观新闻摘要，最多两句话。只陈述事实，不做评价。
  必须包含：核心事件 + 关键数字/产品名/版本号。禁止"值得关注""引发讨论"等评价性套话。
- editor_note: 80-120 字的编辑按语，主观视角。回答"读者为什么应该花时间看这条"——
  可以是判断（这是真突破还是炒作）、对比（和 X 公司同类产品差在哪）、提醒（哪个细节大家都没注意到）。
  禁止重复 chinese_summary 已经讲过的事实。
```

### 验证
- 跑完 main.py 后随便挑 5 条非 GitHub 源 item，肉眼看 `chinese_summary` 是否在 60-80 字范围、`editor_note` 是否角度独立不重复事实。
- 不需要写测试。

## 任务 3：article_fetcher.py 量子位头部清理

**文件**：`src/processors/article_fetcher.py`

### 背景
之前已经加过 `_clean_article_footer()` 砍量子位末尾"版权所有 / 相关推荐"，应用在 cache 加载路径和新抓取路径两处。

现在要加对称的 `_clean_article_header()`——量子位每篇开头都有固定 byline 模板：

```
Jim Fan全新暴论出炉      ← 文章副标题（每篇不同，无法死匹配）
henry 发自 凹非寺        ← "XX 发自 YY"，签发地点
量子位 | 公众号 QbitAI    ← 固定媒体署名
```

### 设计

**仅对 qbitai.com 域名生效**（用户明确选 Q4=A，不要做通用启发式，避免误伤其他源）。

判定逻辑（建议）：
1. 检查 item 的 url / domain 是否包含 `qbitai`（量子位域名 `qbitai.com`）
2. 把正文按行切（trafilatura 给的是 markdown，段落用 `\n\n` 分隔）
3. 在前 5 段范围内查找如下任一 marker，找到就把**包含 marker 的那段以及它之前的所有段**全部砍掉：
   - 正则 `r"^\S+\s*发自\s*\S+$"`（匹配"henry 发自 凹非寺"这种形态）
   - 字面包含 `"公众号 QbitAI"` 或 `"公众号QbitAI"`
   - 字面等于 `"量子位"`（独占一段）
4. 取最靠后那个 marker 的位置作为切点（避免漏砍中间一行）

### 实现位置
和 `_clean_article_footer` 同级，加 `_clean_article_header(text: str, source_url: str) -> str`。**两个 clean 函数都要在以下两处调用**：
1. `_load_cache()` 里——历史缓存的文章不会重新抓，必须在加载时清
2. 新抓取路径里 trafilatura 拿到 content 后立即清

参考之前 `_clean_article_footer` 的模式，**不要改 footer 的逻辑**——它是独立的，header 是新的。

### 验证
1. 删 `data/article_cache/` 里量子位的 cache（或直接删整个 cache 目录，反正有备份）—— 否则历史缓存里旧的"含 byline 头"的版本不会触发新逻辑。等等，按我们的设计 `_load_cache` 也清，所以**理论上不需要删 cache**，但稳妥起见还是建议删一下重抓。
2. 删 `data/daily/2026-05-09.json` 和 `web/data/2026-05-09.json`
3. 重跑 main.py（**Windows 必须 `PYTHONUNBUFFERED=1 python -u src/main.py 2>&1`**，否则后台跑 stdout 会被 block-buffered，CLAUDE.md 暂时还没写这点但后续要补）
4. 端到端约 12-15 分钟
5. 验证：在 `web/data/2026-05-09.json` 里抽 3 条 source=量子位 / qbitai 的 item，看 `content` 字段开头是不是直接进入正文（"VLA已死，世界动作模型当立..."），**不再有** "Jim Fan全新暴论 / henry 发自 / 公众号 QbitAI" 三行
